Phase 1: Systematische Analysemethoden

Die 3 Schritte der Komplexitätsanalyse

1. Grundoperationen identifizieren
2. Operationen zählen (abhängig von n)
3. Best-, Average-, Worst-Case bestimmen

Schritt 1: Grundoperationen

Frage: Was wird am häufigsten ausgeführt?
  • Bubble Sort: Vergleiche + Vertauschungen
  • Selection Sort: Vergleiche (Minimum-Suche)
  • Insertion Sort: Vergleiche + Verschiebungen

Meist dominieren die Vergleichsoperationen

Schritt 2 & 3: Zählen und Fälle unterscheiden

Best Case: Günstigste Eingabe
Average Case: Durchschnittliche Eingabe
Worst Case: Ungünstigste Eingabe

Übung: Welche Eingabe ist für Bubble Sort am günstigsten?

Phase 2: Vergleichende Bewertung

Komplexitäts-Vergleichstabelle

Algorithmus Best Case Average Case Worst Case
Bubble Sort O(n) O(n²) O(n²)
Selection Sort O(n²) O(n²) O(n²)
Insertion Sort O(n) O(n²) O(n²)

Weitere Eigenschaften

Algorithmus Speicher Stabil? In-Place?
Bubble Sort O(1) Ja Ja
Selection Sort O(1) Nein Ja
Insertion Sort O(1) Ja Ja

Was bedeutet "stabil"?

Stabil: Reihenfolge gleicher Elemente bleibt erhalten

Beispiel: Sortierung von Studenten nach Note

Vorher: Anna(2.0), Bob(1.0), Clara(2.0)
Stabil: Bob(1.0), Anna(2.0), Clara(2.0)
Instabil: Bob(1.0), Clara(2.0), Anna(2.0)

Phase 3: Praxisbezug und Anwendungsszenarien

Wann welcher Algorithmus?

Kleine Datenmengen (n < 50):
→ Insertion Sort (einfach und effizient)
Speicher kritisch:
→ Alle drei geeignet (alle O(1) Speicher)
Teilweise sortierte Daten:
→ Insertion Sort oder optimierter Bubble Sort

Realistische Szenarien

  • Embedded Systems: Insertion Sort für kleine Arrays
  • Online-Algorithmen: Insertion Sort für eingehende Daten
  • Lehrzwecke: Bubble Sort (einfach zu verstehen)
  • Minimale Vertauschungen: Selection Sort

In der Praxis: Meist Hybridansätze (z.B. Quicksort + Insertion Sort)

Ausblick: Effizientere Verfahren

Merge Sort: O(n log n) - immer!
Quick Sort: O(n log n) average, O(n²) worst
Heap Sort: O(n log n) - immer!

Frage: Warum sind O(n log n) Algorithmen besser als O(n²)?

Beispiel: n=1000 → n² = 1.000.000 vs. n log n ≈ 10.000
