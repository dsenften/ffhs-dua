Phase 1: Grundlagenwiederholung

O-Notation Recap

  • O(1) - Konstante Zeit
  • O(n) - Lineare Zeit
  • O(n²) - Quadratische Zeit
  • O(log n) - Logarithmische Zeit

Welche Komplexitäten kennen Sie bereits?

Was bedeutet "sortiert"?

Unsortiert: [3, 1, 4, 2, 5]
Sortiert: [1, 2, 3, 4, 5]

Diskussion:

  • Welche Sortierverfahren kennen Sie?
  • Wo begegnen Ihnen sortierte Daten?

Warum ist Sortieren wichtig?

  • Suchen wird effizienter (Binärsuche)
  • Datenbanken nutzen sortierte Indizes
  • Algorithmen arbeiten oft mit sortierten Daten
  • Benutzerfreundlichkeit (alphabetische Listen)

Sortieren ist ein Grundbaustein der Informatik!

Phase 2: Bubble Sort

Grundidee

Prinzip: Grosse Elemente "blubbern" nach rechts
  • Vergleiche benachbarte Elemente
  • Vertausche, wenn in falscher Reihenfolge
  • Wiederhole für alle Positionen

Bubble Sort - Beispiel

Start: [3, 1, 4, 2]
Schritt 1: [1, 3, 4, 2] (3↔1)
Schritt 2: [1, 3, 4, 2] (3<4)
Schritt 3: [1, 3, 2, 4] (4↔2)
...

Beobachtung: Nach jedem Durchlauf ist das grösste Element am Ende positioniert.

Bubble Sort - Komplexität

Best Case: O(n) - bereits sortiert
Average Case: O(n²) - zufällige Reihenfolge
Worst Case: O(n²) - umgekehrt sortiert

Speicher: O(1) - in-place Sortierung

Phase 3: Selection Sort

Grundidee

Prinzip: Finde das Minimum und tausche es an die richtige Position
  • Suche das kleinste Element
  • Vertausche es mit dem ersten Element
  • Wiederhole für Rest-Array

Selection Sort - Beispiel

Start: [3, 1, 4, 2]
Min=1: [1, 3, 4, 2] (1↔3)
Min=2: [1, 2, 4, 3] (2↔3)
Min=3: [1, 2, 3, 4] (3↔4)
Fertig: [1, 2, 3, 4]

Beobachtung: Linker Teil ist immer sortiert und wird schrittweise erweitert.

Selection Sort - Komplexität

Alle Fälle: O(n²)

  • Warum immer O(n²)?
  • Minimum-Suche benötigt immer n-1, n-2, n-3, ... Vergleiche
  • Unabhängig von der Eingabereihenfolge

Speicher: O(1) - in-place Sortierung

Phase 4: Insertion Sort

Grundidee

Analogie: Wie beim Kartensortieren während des Spiels
  • Nehme nächstes Element
  • Füge es an der richtigen Stelle ein
  • Verschiebe andere Elemente nach rechts

Insertion Sort - Beispiel

Start: [3, 1, 4, 2]
Einfügen 1: [1, 3, 4, 2]
Einfügen 4: [1, 3, 4, 2] (bereits richtig)
Einfügen 2: [1, 2, 3, 4]
Fertig: [1, 2, 3, 4]

Beobachtung: Linker Teil bleibt immer sortiert.

Insertion Sort - Komplexität

Best Case: O(n) - bereits sortiert
Average Case: O(n²)
Worst Case: O(n²) - umgekehrt sortiert

Praktisch relevant: Sehr effizient für kleine oder teilweise sortierte Arrays!
