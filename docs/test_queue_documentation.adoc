== Übersicht

Diese Dokumentation beschreibt die Testfälle für die Queue-Implementierung im `algs4.fundamentals.queue`-Modul.

== Testfälle

Die Testfälle decken folgende Aspekte der Queue-Implementierung ab:

=== Grundfunktionen

==== Leere Queue (`test_empty_queue`)

Überprüft, ob eine neu erstellte Queue leer ist:

* `is_empty()` gibt `True` zurück
* `size()` gibt `0` zurück
* `len()` gibt `0` zurück

==== Enqueue und Dequeue (`test_enqueue_dequeue`)

Testet die grundlegenden Operationen zum Hinzufügen und Entfernen von Elementen:

* Elemente werden in der richtigen Reihenfolge hinzugefügt
* Die Grösse der Queue wird korrekt aktualisiert
* Elemente werden in der FIFO-Reihenfolge entfernt
* Nach dem Entfernen aller Elemente ist die Queue leer

=== Hilfsfunktionen

==== Peek-Funktion (`test_peek`)

Überprüft die Funktionalität der `peek()`-Methode:

* Gibt das erste Element zurück, ohne es zu entfernen
* Nach dem Entfernen des ersten Elements gibt `peek()` das nächste Element zurück

==== String-Repräsentation (`test_repr`)

Testet die String-Repräsentation der Queue:

* Die Elemente werden in der richtigen Reihenfolge dargestellt
* Das Format entspricht den Erwartungen

=== Iterator-Funktionalität

==== Iterator-Test (`test_iterator`)

Überprüft die Iterator-Funktionalität der Queue:

* Die Iteration erfolgt in der richtigen Reihenfolge (FIFO)
* Die Queue bleibt nach der Iteration unverändert

=== Ausnahmefälle und Fehlerbehandlung

==== Dequeue auf leerer Queue (`test_exception_dequeue_empty`)

Testet, ob eine `NoSuchElementException` geworfen wird, wenn `dequeue()` auf einer leeren Queue aufgerufen wird.

==== Peek auf leerer Queue (`test_exception_peek_empty`)

Testet, ob eine `NoSuchElementException` geworfen wird, wenn `peek()` auf einer leeren Queue aufgerufen wird.

=== Verschiedene Datentypen

==== Verschiedene Datentypen (`test_different_types`)

Überprüft, ob die Queue mit verschiedenen Datentypen funktioniert:

* Integer
* String
* Listen

=== FIFO-Reihenfolge

==== FIFO-Reihenfolge (`test_fifo_order`)

Testet, ob die Queue die FIFO-Reihenfolge (First-In-First-Out) einhält:

* Elemente werden in der gleichen Reihenfolge entfernt, in der sie hinzugefügt wurden

==== Hinzufügen nach Entfernen (`test_enqueue_after_dequeue`)

Überprüft, ob das Hinzufügen von Elementen nach dem Entfernen korrekt funktioniert:

* Die Queue behält die FIFO-Reihenfolge bei
* Die interne Struktur bleibt konsistent

== Testausführung

Die Tests können mit dem Python-Unittest-Framework ausgeführt werden:

[source,bash]
----
# Ausführung der Tests im tests-Verzeichnis
python -m unittest tests.test_queue

# Alternativ mit pytest
python -m pytest tests/test_queue.py -v
----

== Testabdeckung

Die Tests decken folgende Aspekte der Queue-Implementierung ab:

* Grundlegende Operationen (enqueue, dequeue)
* Hilfsfunktionen (is_empty, size, peek)
* Iterator-Funktionalität
* Ausnahmefälle und Fehlerbehandlung
* Verschiedene Datentypen
* FIFO-Reihenfolge
* Konsistenz der internen Struktur

== Best Practices

Die Tests folgen diesen Best Practices für Python 3:

* Verwendung des `unittest`-Frameworks
* Klare Testfallnamen, die den Zweck des Tests beschreiben
* Aussagekräftige Docstrings für jeden Test
* Verwendung von `setUp` für die Initialisierung vor jedem Test
* Überprüfung von Ausnahmen mit `assertRaises`
* Tests für verschiedene Datentypen mit generischen Typen
* Überprüfung der Konsistenz der internen Struktur
* Vollständige Abdeckung aller öffentlichen Methoden
