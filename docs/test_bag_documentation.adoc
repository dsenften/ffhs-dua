== Übersicht

Diese Dokumentation beschreibt die Testfälle für die Bag-Implementierung im `algs4.fundamentals.bag`-Modul.

== Testfälle

Die Testfälle decken folgende Aspekte der Bag-Implementierung ab:

=== Grundfunktionen

==== Leere Bag (`test_empty_bag`)

Überprüft, ob eine neu erstellte Bag leer ist:

* `is_empty()` gibt `True` zurück
* `size()` gibt `0` zurück
* `len()` gibt `0` zurück

==== Add-Operation (`test_add`)

Testet die grundlegende Operation zum Hinzufügen von Elementen:

* Elemente werden korrekt hinzugefügt
* Die Grösse der Bag wird korrekt aktualisiert
* Alle hinzugefügten Elemente sind in der Bag enthalten

=== Hilfsfunktionen

==== Size und Len (`test_size_and_len`)

Überprüft die Funktionalität der `size()` und `__len__()` Methoden:

* Beide Methoden geben die korrekte Anzahl der Elemente zurück
* Die Werte werden nach dem Hinzufügen von Elementen korrekt aktualisiert

==== String-Repräsentation (`test_repr`)

Testet die String-Repräsentation der Bag:

* Die Repräsentation beginnt mit "{" und endet mit "}"
* Alle Elemente sind in der Repräsentation enthalten

=== Iterator-Funktionalität

==== Iterator-Test (`test_iterator`)

Überprüft die Iterator-Funktionalität der Bag:

* Alle hinzugefügten Elemente können durch Iteration abgerufen werden
* Die Bag bleibt nach der Iteration unverändert

=== Verschiedene Datentypen

==== Verschiedene Datentypen (`test_different_types`)

Überprüft, ob die Bag mit verschiedenen Datentypen funktioniert:

* Integer
* String
* Listen

=== Spezialfälle

==== Mehrfaches Hinzufügen des gleichen Elements (`test_add_multiple_same_item`)

Testet, ob das gleiche Element mehrmals hinzugefügt werden kann:

* Die Grösse der Bag wird korrekt aktualisiert
* Das Element ist mehrfach in der Bag enthalten

==== Hinzufügen nach Iteration (`test_add_after_iteration`)

Überprüft, ob nach einer Iteration weitere Elemente hinzugefügt werden können:

* Die Grösse der Bag wird korrekt aktualisiert
* Alle Elemente (vor und nach der Iteration hinzugefügt) sind in der Bag enthalten

== Testausführung

Die Tests können mit dem Python-Unittest-Framework ausgeführt werden:

[source,bash]
----
# Ausführung der Tests
python -m unittest tests.test_bag

# Alternativ mit pytest
python -m pytest tests/test_bag.py -v
----

== Testabdeckung

Die Tests decken folgende Aspekte der Bag-Implementierung ab:

* Grundlegende Operationen (add)
* Hilfsfunktionen (is_empty, size, __len__)
* Iterator-Funktionalität
* String-Repräsentation
* Verschiedene Datentypen
* Spezialfälle (mehrfaches Hinzufügen, Hinzufügen nach Iteration)

== Best Practices

Die Tests folgen diesen Best Practices für Python 3:

* Verwendung des `unittest`-Frameworks
* Klare Testfallnamen, die den Zweck des Tests beschreiben
* Aussagekräftige Docstrings für jeden Test
* Verwendung von `setUp` für die Initialisierung vor jedem Test
* Tests für verschiedene Datentypen mit generischen Typen
* Überprüfung der Konsistenz der internen Struktur
* Vollständige Abdeckung aller öffentlichen Methoden
